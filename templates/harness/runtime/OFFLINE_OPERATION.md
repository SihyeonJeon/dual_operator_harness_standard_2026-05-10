# Offline Operation

Status: template

## Works Without Network

- Reading and updating file-backed harness context.
- Local planning and task blueprint drafting.
- Local validation scripts copied into the target project.
- Local approval packet drafting and review.
- Local audit records, state files, and evaluation reports.
- Local static HTML status reports generated by `python3 scripts/harnessctl.py report`.
- Local pre-visualization spec checks generated by
  `python3 scripts/harnessctl.py viz-spec-check`.

## Requires Network, Credentials, Or External Runtime

- Public web research.
- Browser/computer-use.
- Account login.
- API-key use.
- Customer outreach.
- Public posting.
- Marketplace or proposal submission.
- Payment, invoice, refund, contract action, or movement of money.
- Cloud deployment or always-on worker execution.
- Remote terminal control or mobile approval channels.

## Local Machine Powered Off

If the local machine is powered off, local-only workers cannot continue. Any
always-on operation requires a separately approved cloud runner, secret policy,
approval channel, audit store, budget, and smoke evidence.

Mobile or remote operation must be treated as an approved connector over a
bounded runner, not as unrestricted terminal access. See
`harness/runtime/REMOTE_OPERATION_POLICY.md`.

## Control Rule

Offline/local-continuity does not grant permission for external actions. All
network writes, financial actions, account actions, and cloud execution remain
denied until explicitly approved and verified.
