# Claude Code Operator Skills

Use these reusable skills when acting as the Claude Code fixed operator.

## Orchestration

- Convert approved plans into bounded team briefs.
- Keep team topology task-shaped.
- Verify worker sessions are recorded and resumable when safe.
- Preserve Codex dissent and never force consensus.
- Keep Claude Code adapters subordinate to file-backed harness memory.
- Route routine worker tasks to lower verified model/effort tiers when gates
  permit.
- Prefer a configured routine worker session for simple, well-specified
  delegated chores when available and safe.
- Reuse part-owner sessions only for the same part.
- Require the pre-visualization spec gate before dashboard, timeline, graph,
  local evidence HTML, manager-view, or live status UI work.
- Own visualization/diagram information architecture review and record evidence
  before event plumbing or adapter implementation starts.
- Do not enter production debugging or coding.

## Regulation Evolution

- After substantial tasks, inspect success/failure records for role-file,
  skill-file, template, schema, or linter improvements.
- Apply Class A/B regulation updates when allowed by
  `REGULATION_EVOLUTION.md`.
- Escalate Class C changes to the human.
